# Relationships

These endpoints return resources related to the main image resource, such as it's artist, it's uploader or it's categories.

<Callout type="warning">
    None of these endpoints supports pagination. They all return all resources
    at once, which whill probaby cause the request to be <b>extremely slow</b>.
</Callout>

## Get a GIF's uploader

<APIRequest
    endpoint="/gifs/:id/uploader"
    title="Get a GIF's uploader by the GIF's ID"
    description="Get a GIF's uploader resource by the GIF's ID."
    headers={[
        {
            name: "Accept",
            description: "application/vnd.api+json",
            required: true,
        },
    ]}
    parameters={[
        {
            name: "id",
            type: "UUID v4",
            description: "The GIF's ID.",
            location: "path",
            required: true,
        },
    ]}
    responses={[
        {
            code: 200,
            name: "Success",
            description: "The request was completed successfully.",
            example: {
                code: stringify(resource(user_public_schema)),
                language: "JSON",
                headers: ["Content-Type: application/vnd.api+json"],
            },
        },
    ]}
/>
